# 灵感引擎 — 人工手动测试文档

> 版本: v1.2 | 日期: 2026-05-12 | 更新: 新增 Agent 核心能力优化相关测试用例（错误恢复、防循环保护、并行执行、优雅降级、结果管理、增量摘要、Prompt 缓存、Token 追踪）

## 测试前准备

### 环境要求

| 项目 | 要求 |
|------|------|
| 后端 | 已启动 (`uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload`) |
| 前端 | 已启动 (`npm run dev`，默认 `http://localhost:5173`) |
| 数据库 | PostgreSQL 已运行，已执行 `alembic upgrade head` |
| 配置 | `.env` 文件中各 API Key 已正确配置 |
| 开发模式 | `BYPASS_OAUTH_LOGIN=true`（跳过知乎 OAuth 登录） |
| 记忆系统 | `MEMORY_AUTO_REVIEW=true`, `MEMORY_NUDGE_INTERVAL=8`, `MEMORY_CHAR_LIMIT=2200`, `USER_CHAR_LIMIT=1375` |
| 错误恢复 | `MAX_EMPTY_RETRIES=3`, `MAX_STREAM_RETRIES=2` |
| 工具防护 | `TOOL_MAX_REPEAT=3`, `TOOL_MAX_CONSECUTIVE_ERRORS=5` |
| 结果管理 | `TOOL_RESULT_INLINE_LIMIT=3000`, `TOOL_RESULT_PERSIST_THRESHOLD=8000`, `TURN_TOTAL_BUDGET=15000` |
| 并行执行 | `PARALLEL_TOOL_ENABLED=true` |

### 测试工具

- 浏览器（Chrome / Edge 推荐）
- 终端（用于 curl 命令行测试 API）
- 可选：Postman / HTTPie

### 环境检查清单

- [ ] 执行 `curl --noproxy localhost http://localhost:8001/api/health`，确认返回 `{"status":"ok"}`
- [ ] 打开 `http://localhost:5173`，确认前端页面加载正常
- [ ] 确认 PostgreSQL 连接正常（后端启动日志无数据库错误）

## 已知限制

1. **知乎 API 调用频率限制**：热榜接口有频率限制，短时间内反复请求可能被限流
2. **MiniMax API 依赖**：所有 AI 对话功能依赖 MiniMax API 可用性
3. **HMAC 签名时效**：知乎社区 API 的 HMAC 签名有时间窗口限制（±300 秒）
4. **自动 Skill 提炼**：依赖 LLM 判断质量，不保证每次都有产出
5. **SQLite ARRAY 兼容性**：卡片的 `tags` 字段使用 PostgreSQL ARRAY 类型，本地测试需要 PostgreSQL
6. **MemoryReviewer 异步延迟**：记忆自动回顾在会话结束后异步执行，检查结果需等待 5-10 秒
7. **Memory Nudge 非确定性**：Nudge 提醒注入 system prompt 后，Agent 是否调用 memory 工具取决于 LLM 的判断
8. **记忆字符限制**：`MEMORY.md` 默认 2200 字符、`USER.md` 默认 1375 字符，超限时新条目将被拒绝
9. **Prompt 缓存依赖 Provider**：`cache_control` 前缀缓存仅在 Anthropic 兼容 API 下生效，ChatCompletions 模式自动退化为字符串拼接
10. **并行执行限于只读工具**：写操作工具（memory、skill_manage、publish）始终顺序执行，仅 10 个只读工具支持并行
11. **流式重试有状态丢失**：流式传输断开重试时，已接收的部分文本/工具调用数据会丢失（重试从头开始）
12. **工具结果持久化目录增长**：大工具输出持久化到 `data/tool_outputs/`，长期使用需定期清理
