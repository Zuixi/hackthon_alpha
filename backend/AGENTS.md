# Backend Agent Guide（精简版）

后端目标：提供认证、热点、对话、卡片、发布五类能力；统一外部接口调用；保证数据与鉴权安全。

## 分层约束

- 启动层：应用初始化、路由挂载、中间件与健康检查。
- 配置层：环境变量与默认值统一管理。
- 认证层：OAuth 登录、JWT 签发与用户鉴权。
- 路由层：只做参数校验与编排，不写第三方调用细节。
- 服务层：封装知乎与 AI 调用，统一超时、错误与降级。
- 数据层：模型关系、迁移、事务与查询作用域控制。

## 业务规则

- 认证：回调参数先规范化；默认 Bearer 鉴权；开发态可免 OAuth。
- 热点：优先外部接口，失败回退本地数据；使用缓存控制请求频率。
- 对话：先落用户消息，再流式返回，再落助手消息；SSE 事件语义保持稳定。
- 卡片：归属当前用户，支持标签/搜索/分页与标准 CRUD。
- 发布：依赖用户令牌，令牌缺失快速失败，成功返回可展示结果。

## 变更原则

- 先理解边界，再改代码；小步、低耦合、可回滚。
- 新增能力时，同步补齐契约、鉴权、错误语义、数据关系。
- 查询默认限制当前用户作用域，避免越权。
- 外部调用必须有超时、日志和可理解错误提示。

## 完成标准

- 职责边界不漂移，接口风格与现有模块一致。
- 关键链路完成最小验证（启动/接口可用/主流程）。
- 经验同步到变更记录与项目记忆。
