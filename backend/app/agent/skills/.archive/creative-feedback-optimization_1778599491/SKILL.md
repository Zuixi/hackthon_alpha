## 工作流程

### 第一步：生成初始方案
根据用户需求快速生成多个创意选项，每个选项包含：
- 核心设定描述
- 主要爽点/看点
- 基本的故事框架

### 第二步：收集用户反馈
等待用户对各方案的点评，记录用户在以下维度的评价：
- 创作难度（设定复杂度、逻辑要求）
- 阅读体验（是否烧脑、是否轻松）
- 场景转换（是否单一、是否多元）
- 核心方向认可度

### 第三步：提炼反馈要点
将用户的负面反馈转化为优化原则：
- ❌ → 创作/阅读门槛过高 → 降低复杂度
- ❌ → 场景单一 → 设计多元场景转化
- ❌ → 逻辑过重 → 简化逻辑，轻松阅读
- ✅ → 某方向可行 → 保留并强化

### 第四步：制定优化策略
总结新的核心原则，确保新方案满足：
1. 设定简单，展开丰富
2. 场景转化自然
3. 阅读轻松，爽点密集
4. 保留有效方向，调整不足元素

### 第五步：生成优化方案
基于新原则重新生成2-3个方案，每个方案：
- 解决原有问题
- 保留有效元素
- 包含爽点设计和场景推进方向

### 第六步：展示与选择
呈现新方案，引导用户选择或进一步调整
