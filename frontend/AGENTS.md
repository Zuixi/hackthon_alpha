# Frontend Agent Guide（精简版）

前端目标：提供登录、热点浏览、创作对话、灵感卡片、发布操作的完整体验；保证状态一致、交互稳定、错误可恢复。

## 分层约束

- 应用层：路由、全局 Provider、基础运行配置统一管理。
- 鉴权层：登录态判定、用户信息获取、退出清理与未授权跳转。
- 接口层：统一请求封装、Token 注入、错误处理、401 兜底。
- 状态层：服务端状态统一由 React Query 管理，避免重复状态源。
- 视图层：页面负责交互编排，通用组件负责样式与复用。
- 类型层：接口结构与前后端字段语义保持一致。

## 业务规则

- 登录：从授权地址跳转，回调后写入登录凭证并进入主流程。
- 导航：受保护页面必须在登录态下访问，未登录统一回登录页。
- 热点：列表查询走缓存策略，支持手动刷新和后续创作入口。
- 对话：会话列表与详情分离；发送消息走流式链路并兼容增量事件。
- 卡片：支持新建、检索、标签筛选、删除与发布，操作后刷新相关缓存。
- 发布：支持从对话消息和卡片内容触发，结果要有明确反馈。

## 变更原则

- 先确认用户流与数据流，再做 UI 或逻辑改动。
- 所有写操作后要正确失效/刷新查询，避免界面与后端不一致。
- 流式对话改动必须保持事件解析兼容性，避免历史会话展示异常。
- 错误处理优先“用户可理解 + 可恢复”，避免静默失败。

## 完成标准

- 页面流程可达且无登录态断裂（登录、进入、退出）。
- 关键链路可用（热点拉取、对话流式、卡片增删、发布反馈）。
- 新增或调整的行为已同步到变更记录与项目记忆。
